Clases

Las clases, son unos de los elementos que mas importancia tienen en la Programacion orientada a objetos, ya que permite la abstraccion del mundo real, que permiten el encapsulamiento de datos y compprtamientos relacionados.

    Caracteristicas

  • Atributos: Las clases tienen atributos que representan datos asociados con la clase. Estos son variables que almacenan información sobre el estado del objeto.
  • Metodos: Las clases contienen métodos que representan el comportamiento del objeto. Estos son funciones que definen lo que el objeto puede hacer.
  • Encapsulamiento: Las clases permiten encapsular sus atributos y métodos, lo que significa que puedes ocultar ciertos detalles internos y exponer solo lo necesario para interactuar con el objeto. Esto mejora la modularidad y la seguridad del código
  • Herencia: Las clases pueden heredar atributos y métodos de otras clases. Esto promueve la reutilización del código y la creación de jerarquías de clases
  • Instanciacion: Para utilizar una clase, se crea una instancia u objeto de esa clase. Una instancia representa un único ejemplar de la clase.
  • Construccion: Las clases pueden tener un método especial llamado constructor, que se ejecuta automáticamente cuando se crea una nueva instancia de la clase. Se utiliza para inicializar los atributos del objeto.
  • Destructor: Al igual que el constructor, algunas programaciones orientadas a objetos admiten un destructor que se ejecuta cuando un objeto se elimina o sale de ámbito
  • Abstraccion: Las clases permiten la abstracción al modelar objetos del mundo real y sus interacciones, simplificando la representación del problema.
  • Poliformismo : Las clases pueden tener múltiples formas (polimorfismo). Esto significa que varias clases pueden implementar métodos con el mismo nombre pero con comportamientos específicos para cada clase.

Objetos

Los objetos son instancias especificas de una clase orientada a objetos

    caracteristicas

  • Identidad Unica: Cada objeto tiene una identidad única, lo que significa que es una instancia específica de una clase y puede ser diferenciado de otros objetos.
  • Estado: Los objetos tienen un estado representado por sus atributos. Estos atributos almacenan la información sobre las características del objeto en un momento dado.
  • Comportamiento: Los objetos pueden realizar acciones o tener comportamientos definidos por los métodos asociados a su clase. Estos métodos definen lo que un objeto puede hacer.
  • Encapsulamiento: Los objetos encapsulan su estado y comportamiento, lo que significa que pueden ocultar ciertos detalles internos y exponer solo lo necesario para interactuar con ellos. Esto mejora la modularidad y la seguridad del código.
  • Instanciacion: Un objeto es creado mediante el proceso de instanciación, que implica tomar una clase y crear un ejemplar específico de esa clase
  • Relacion con clases: Los objetos están vinculados a una clase específica y se crean utilizando la plantilla proporcionada por esa clase. Comparten los atributos y métodos definidos en la clase
  • Herencia: Los objetos pueden heredar atributos y métodos de clases base. Esto promueve la reutilización del código y la creación de jerarquías de clases.
  • Poliformismo: Los objetos pueden tener múltiples formas (polimorfismo). Esto significa que varias clases pueden tener métodos con el mismo nombre pero con implementaciones específicas para cada clase.
  • Duracion de vida: Los objetos tienen una duración de vida que comienza cuando se crean y termina cuando se destruyen o se eliminan.
  • Interaccion: Los objetos pueden interactuar entre sí enviándose mensajes o llamando a los métodos de otros objetos
