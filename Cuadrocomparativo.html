Cuadro Comparativo

Caracteristicas

  • Identidad Unica
  • Estado
  • Comportamiento
  • Encapsulamiento
  • Instanciacion
  • Relacion con clases
  • Herencia
  • Poliformismo
  • Duracion de vida
  • Interaccion

Clases

  • Representa plantilla general o tipo
  • Define atributos comunes para todas las instancias
  • Contiene los metodos compartidos por todas las instancias
  • Oculta detalles internos y expone interfaz publica
  • No se instancia directamente, solo se utiliza como plantilla
  • Puede tener relaciones de herencia y composicion
  • Puede heredar atributos y metodos de otras clases
  • Permite la creacion de metodos con el mismo nombre en clases derivadas
  • Existe durante todo el ciclo de vida de la aplicacion
  • Puede interactuar con otras clases objetos

Objetos

  • Cada objeto es una clase unica de una clase
  • Almacena informacion especifica de la instancia
  • Realiza acciones especificas segun su clase
  • Encapsula su estado y comportamiento
  • Creado mediante el proceso de instanciación
  • Esta vinculado a una clase especifica
  • Puede heredar caracteristicas de su clase base
  • Puede tener multiples formas y comportamientos
  • Tiene una duracion de vida limitada, se puede crear o destruir
  • Puede enviar mensajes o llamar a metodos a otros objetos
Este cuadro comparativo resalta las diferencias y similitudes entre clases y objetos, destacando cómo las clases proporcionan la estructura y plantilla, mientras que los objetos representan instancias específicas y encapsulan datos y comportamientos únicos. Además, señala cómo ambos pueden interactuar entre sí y con otros elementos del sistema.
